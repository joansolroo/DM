                  dovyski Stuart Fyfe 5 months ago

                  Thanks! Regarding your code, the following line is wrong:

                  desired *= (self.MAX_VELOCITY * distance / self.slowingRadius)

                  It should be replaced with:

                  desired *= self.MAX_VELOCITY * (distance / self.slowingRadius)

                  The piece (distance / self.slowingRadius) is what makes the boid stop, because its value will always be between 0 and 1 (the closer the boid is to the target, the closer this piece is zero). In your current code, distance is multiplied by MAX_VELOCITY before it is divided by slowingRadius, which will not produce the expected value between 0 and 1 (in fact it will always be greater than 1, that's why the boids never stop).

                          Sorry for this late reply! Hope it's still helpful...

                          The truncate() function takes two values, a vector and a number ("max"), and returns a vector that is in the same direction as the input vector, but is no longer than "max". (If the input vector is already shorter than "max", then the output vector is the same as the input vector.)

                          The code for the function is a little dense, and assumes knowledge of the ternary operator: http://www.as3blog.org/2014/04...

                          In this case, Fernando uses truncate() to ensure that the length of the velocity vector is never greater than "max_speed".

                                                            dovyski GladDukat . a year ago

                                                            I'm glad you like it! About your questions:

                                                            1. Actually there is no calculation related to the slowing radius, it is just a number I use to slow down entities. The way it was implemented, it is not related to the entity velocity, so an entity moving really fast will not be able to slow down and stop before reaching the circle center.

                                                            2. acceleration/decelleration acts upon velocity, but it does not change/command max velocity. Just like in real world, acceleration is a factor that increases an entities velocity over time: something like v = v + a, where "v" is velocity and "a" is acceleration. If you assume the entity has no velocity limits, the calculation v = v + a will be performed every game update, so eventually v will increase until it reaches "infinity". That's not true in real world (nor gamedev-world). Usually entities have limitations, like the power limit of a car engine. At some point, the car power will not be enough to overcome all forces acting against it (engine friction, air friction, etc), so the entity's velocity will reach "max velocity".

                                                            In that case, the acceleration remains the same, but it will not affect the car's velocity anymore. Is it clear now why I used max velocity?

                                                                  A quick note: in your slowing down code for Arrive you are normalizing the desired velocity then multiplying by its length--these operations are cancelling each other out. So instead of

                                                                  desired_velocity = normalize(desired_velocity) * max_velocity * (distance / slowingRadius)

                                                                  you could simply do:

                                                                  desired_velocity = desired_velocity * (max_velocity / slowingRadius)

                                                                  cheers!
